# 금욕 프로젝트 인수인계 문서

작성일: 2026-05-30
작성 목적: iOS 앱 `금욕`을 다른 개발자나 디자이너가 이어받아도 현재 상태, 의도, 구조, 남은 일을 빠르게 이해할 수 있도록 정리한다.

---

## 1. 프로젝트 한 줄 요약

`금욕`은 금연, 야식 끊기, SNS 줄이기처럼 사용자가 스스로 정한 절제 목표를 만들고, 며칠 동안 성공했는지 매일 기록하는 SwiftUI 기반 iPhone 앱이다.

앱의 핵심은 복잡한 습관 관리가 아니라 `오늘 성공했는가`, `며칠째 이어가고 있는가`, `실패하거나 완료한 금욕을 다시 볼 수 있는가`에 집중하는 것이다.

---

## 2. 제품 방향성

사용자가 원하는 최종 방향은 다음과 같다.

- 최대한 쓸데없는 요소를 빼고 본질만 남긴다.
- Apple 순정 앱처럼 자연스럽고 익숙한 인터랙션을 우선한다.
- 커스텀 제스처나 과한 애니메이션보다 SwiftUI / UIKit 기본 동작을 우선한다.
- 기능을 무리하게 늘리기보다 기록의 신뢰감, 삭제/실패 같은 위험 행동의 안정성, 화면의 단순함을 중시한다.
- 감성적인 앱이되, 과한 문구나 복잡한 설명은 줄이는 방향이 좋다.

향후 개발 시 중요한 원칙:

- 기본 List, NavigationStack, sheet, confirmationDialog, swipeActions, EditMode 같은 Apple 기본 패턴을 먼저 검토한다.
- 사용자가 명확히 요청하지 않는 한 커스텀 드래그 선택, 커스텀 피커, 과한 화면 전환은 피한다.
- 앱스토어 배포를 목표로 하므로 실제 기기 테스트와 접근성 확인이 필요하다.

## 4. 현재 Git 작업 상태

현재 `main`은 `origin/main`과 동기화되어 있다.

다만 로컬에는 커밋하지 않은 변경이 있다.

```text
M Geumyok.xcodeproj/project.pbxproj
?? Geumyok.etc/
```

### 4.1 project.pbxproj 변경

이 변경은 앱 기능 수정이 아니라 Xcode가 건드린 프로젝트 설정 변경이다.

현재 diff 내용:

- `objectVersion`이 `56`에서 `60`으로 변경됨
- `DEVELOPMENT_TEAM = S4F84Q6XD7`가 Debug / Release 설정에 추가됨

이 변경은 지금까지 기능 커밋에 일부러 포함하지 않았다. Apple Developer Team / Xcode 버전 관련 변경일 수 있으므로, 실제 배포 설정으로 확정할 때만 커밋 여부를 판단해야 한다.

### 4.2 Geumyok.etc 폴더

현재 Git에는 추적되지 않는 폴더다.

```text
Geumyok.etc/Geumyok app icon.png
Geumyok.etc/Geumyok introduction (korean.ver).txt
Geumyok.etc/.DS_Store
```

이 폴더에는 앱 아이콘 원본 이미지와 앱 소개글 텍스트가 들어 있다. 정식으로 보존하려면 `.DS_Store`는 제외하고 필요한 파일만 커밋하는 것을 권장한다.

## 5. 기술 스택

- Swift
- SwiftUI
- UIKit 일부 연동
- UserDefaults 로컬 저장
- Xcode iOS 앱 프로젝트

외부 라이브러리는 없다.

## 7. 앱 데이터 모델

### 7.1 DayStatus

```swift
enum DayStatus: String, Codable, Equatable {
    case success
    case failure
}
```

하루 기록이 성공인지 실패인지 나타낸다.

### 7.2 ChallengeOutcome

```swift
enum ChallengeOutcome: String, Codable, Equatable {
    case inProgress
    case completed
    case failed
}
```

금욕 전체의 상태다.

- `inProgress`: 진행 중
- `completed`: 목표 일수 달성
- `failed`: 실패로 종료

### 7.3 DailyCheckIn

```swift
struct DailyCheckIn: Identifiable, Codable, Equatable {
    var id = UUID()
    var date: Date
    var status: DayStatus
}
```

각 날짜별 성공/실패 기록이다.

### 7.4 Challenge

핵심 도메인 모델이다.

필드:

```swift
var id: UUID
var name: String
var startDate: Date
var targetDays: Int
var records: [DailyCheckIn]
var outcome: ChallengeOutcome
var endedDate: Date?
```

계산 속성:

- `isActive`: 진행 중 여부
- `successCount`: 성공 기록 개수
- `progress`: 목표 대비 진행률, 0...1
- `remainingDays`: 남은 목표 일수

이름은 앞뒤 공백 제거 후 최대 24자로 제한된다. 비어 있는 이름은 모델 차원에서는 `금욕`으로 보정되지만, 현재 생성 UI에서는 이름이 비어 있으면 생성하지 않고 알림을 띄운다.

## 8. 저장 구조

저장 방식은 `UserDefaults`다.

현재 저장 키:

```swift
private let storageKey = "geumyok.challenges.v2"
private let legacyStorageKey = "geumyok.challenge.v1"
```

저장 payload:

```swift
private struct ChallengeStorage: Codable {
    var challenges: [Challenge]
    var selectedChallengeID: UUID?
}
```

현재 앱 로드시 `selectedChallengeID`는 일부러 `nil`로 초기화된다. 그래서 앱을 껐다 켜면 항상 홈 목록 화면부터 시작한다.

첫 실행 튜토리얼 여부는 `@AppStorage`로 별도 저장한다.

```swift
@AppStorage("geumyok.hasSeenOnboarding.v1") private var hasSeenOnboarding = false
```

앱을 삭제 후 다시 설치하면 UserDefaults가 초기화되므로 첫 실행 인트로와 튜토리얼이 다시 나온다.

## 9. ChallengeStore 동작 정리

### 9.1 startChallenge(name:targetDays:)

새 금욕을 만든다.

동작:

- 이름이 비어 있으면 실패 반환
- 목표 일수는 1일 이상 365일 이하로 제한
- 시작일은 오늘의 startOfDay
- 상태는 `inProgress`
- 생성 후 해당 금욕을 선택 상태로 만든다
- 같은 이름이 이미 있으면 자동으로 회차 이름을 만든다

예시:

```text
금연
금연 2회차
금연 3회차
```

### 9.2 selectChallenge

목록에서 금욕을 선택했을 때 상세 화면 대상으로 지정한다.

- 진행 중 금욕이면 `selectedChallengeName`에도 이름을 넣는다.
- 종료된 금욕이면 생성 폼 이름에는 영향을 주지 않도록 빈 값으로 둔다.

### 9.3 prepareRestart

실패한 금욕을 다시 시작할 때 사용한다.

- 기존 목표명 기반으로 다음 회차 이름을 미리 채운다.
- 목표 일수도 기존 값으로 미리 채운다.
- 즉시 재시작하지 않고 생성창으로 이동해서 사용자가 수정할 수 있게 한다.

### 9.4 renameSelectedChallenge

상세 화면에서 제목을 눌러 이름을 수정할 수 있다.

- 빈 이름이면 실패
- 최대 24자 제한
- 진행 중이면 생성 폼 상태도 함께 갱신

### 9.5 toggleTodaySuccess

오늘 성공 버튼 동작이다.

- 오늘 성공 기록이 없으면 성공 기록 추가
- 오늘 이미 성공이면 오늘 기록 제거
- 즉 `오늘 성공 수정`은 성공 전 상태로 되돌리는 역할이다.

### 9.6 recordToday(.failure)

실패 기록이다.

현재 UI에서는 바로 실행하지 않고 `confirmationDialog` 확인창을 거친다.

실패 처리 결과:

- 오늘 날짜로 실패 기록 저장
- outcome을 `failed`로 변경
- endedDate를 오늘로 설정
- 선택 상태를 해제해서 홈으로 돌아가게 함

### 9.7 목표 달성

성공 기록 수가 목표 일수 이상이 되면 자동 완료된다.

- outcome을 `completed`로 변경
- endedDate를 오늘로 설정
- 선택 상태를 해제해서 홈으로 돌아가게 함

### 9.8 삭제

단일 삭제와 다중 삭제를 모두 지원한다.

선택 중인 금욕이 삭제되면 `selectedChallengeID`를 nil로 만든다.

## 10. 현재 화면 흐름

### 10.1 앱 시작

ContentView가 루트다.

앱 시작 시 항상 짧은 인트로 오버레이가 뜬다.

첫 설치 후 첫 실행이고, 아직 저장된 금욕이 없고, 튜토리얼을 본 적이 없으면:

1. 검은 배경 위에 예전 스타일 인트로 표시
2. `금욕` 텍스트, 링, flame 아이콘, 보조 문구 표시
3. 인트로가 사라진 뒤 튜토리얼 sheet 표시
4. 튜토리얼을 닫거나 `시작하기`를 누르면 `geumyok.hasSeenOnboarding.v1`이 true가 됨

이미 튜토리얼을 봤거나 기존 기록이 있으면:

1. 짧은 검은 화면만 표시
2. 바로 홈으로 이동

### 10.2 튜토리얼 화면

구성:

- 제목: `처음 시작하기`
- 설명: `목표를 만들고 하루에 한 번만 기록하세요.`
- 안내 3개:
  - 금욕 만들기: 목표명과 목표 일수를 정해요.
  - 오늘 성공: 하루가 끝나면 성공을 기록해요.
  - 지난 금욕: 끝난 기록은 목록에서 다시 볼 수 있어요.
- 버튼: `시작하기`

튜토리얼은 `.medium`, `.large` detent를 가진 sheet다.

### 10.3 홈 화면

홈은 `ChallengeDashboardView`다.

구성:

- 상단 타이틀 `금욕`
- 보조 문구 `오늘 이어갈 금욕을 골라주세요`
- 우측 + 버튼으로 금욕 생성 sheet 열기
- 진행 중 금욕 섹션
- 지난 금욕 섹션

홈 화면에서는 진행 중 / 지난 금욕 각각 최대 5개까지만 보여준다. 5개를 넘으면 더보기 버튼이 나온다.

### 10.4 빈 홈 화면

금욕이 하나도 없으면 `EmptyChallengeList`가 보인다.

문구:

- `아직 금욕이 없어요`
- `금연, 야식 끊기, SNS 줄이기처럼 따로 관리할 목표를 만들어보세요.`
- 버튼: `첫 금욕 만들기`

이 화면은 기존 목록을 모두 삭제했을 때 자연스럽게 나타나도록 상태 전환을 조정해 둔 상태다.

### 10.5 생성 화면

`StartChallengeView`다.

입력 항목:

- 목표명
- 목표 일수

목표명:

- placeholder: `예: 금연, 야식 끊기`
- 비어 있으면 생성하지 않고 alert 표시
- alert 제목: `목표명을 설정해주세요`
- alert 메시지: `이 금욕을 뭐라고 부를지 먼저 정해볼까요?`

목표 일수:

- 숫자 직접 입력 가능
- Apple 기본 휠처럼 보이는 `UIPickerView` 사용
- 숫자 입력 키보드에 `완료` 툴바 버튼 있음
- 1...365 범위로 제한

생성 버튼:

- `금욕 시작하기`

### 10.6 상세 화면

`ActiveChallengeView`다.

상단:

- 목표명
- D+일수와 목표 일수
- 시작일 / 종료 예정일 또는 실제 종료일

중앙:

- 원형 진행 링
- 성공 일수
- 남은 날

진행 중일 때 버튼:

- `오늘 성공`
- 성공 후에는 `오늘 성공 수정`으로 바뀜
- 성공 후에는 실패 버튼 비활성화
- `실패로 기록`은 빨간 글자 계열이며, 누르면 확인창 표시

실패 확인창:

- 제목: `실패로 기록할까요?`
- destructive 버튼: `실패로 기록`
- 취소 버튼: `취소`
- 메시지: `이 금욕은 종료되고 지난 금욕에 저장됩니다.`

종료된 금욕일 때:

- 완료 상태면 `목표 달성`
- 실패 상태면 `다시 시작해볼까요?`
- 실패 상태의 다시 시작 버튼을 누르면 같은 목표명/목표일수로 생성창이 열리고, 이름은 다음 회차로 미리 채워짐

### 10.7 목표명 수정

상세 화면에서 목표명 텍스트를 누르면 키보드 입력 상태로 전환된다.

- UIKit UITextField 기반
- 키보드 툴바에 완료 버튼 있음
- 완료 또는 포커스 해제 시 저장
- 최대 24자 제한

### 10.8 더보기 화면

`ChallengeArchiveView`다.

진행 중 금욕 더보기와 지난 금욕 더보기를 같은 컴포넌트로 처리한다.

중요한 구현 방향:

- Apple 기본 `List(selection:)`과 `EditMode`를 사용한다.
- 이전에 커스텀 드래그 선택을 만들었지만 스크롤과 충돌해서 제거했다.
- 앞으로도 이 부분은 Apple 순정 EditMode 기반을 유지하는 것이 좋다.

동작:

- 우측 상단 `선택` 버튼으로 선택 모드 진입
- 선택 모드에서는 여러 항목 선택 가능
- 하단의 작은 `삭제` 버튼으로 선택 항목 일괄 삭제
- 일반 모드에서는 행을 눌러 상세로 진입
- 일반 모드에서는 swipeActions로 단일 삭제 가능
- full swipe 삭제는 꺼져 있음: `allowsFullSwipe: false`

## 11. UI / 디자인 상태

## 19. 알려진 제약과 주의점

### 19.1 ContentView.swift가 너무 크다

현재 UI 대부분이 한 파일에 들어 있다. 기능이 늘어나면 유지보수가 어려워진다.

우선순위가 생기면 화면별로 분리하는 리팩터링을 추천한다.

### 19.2 테스트 코드가 없다

ChallengeStore의 날짜 계산, 완료 처리, 실패 처리, 회차 이름 생성은 앱의 핵심 로직이다. 추후에는 단위 테스트를 추가하는 것이 좋다.

우선 테스트해야 할 케이스:

- 목표 일수 1일 생성 후 오늘 성공하면 바로 completed가 되는지
- 오늘 성공 후 다시 누르면 기록이 제거되는지
- 실패 기록 시 outcome이 failed가 되는지
- 실패 후 selectedChallengeID가 nil이 되는지
- 같은 이름 생성 시 2회차, 3회차가 붙는지
- 24자 초과 이름이 잘리는지
- 365일 초과 목표가 365로 제한되는지
- 저장 후 앱 재실행 시 목록이 복원되는지

### 19.3 iCloud 동기화 없음

현재 데이터는 기기 로컬 UserDefaults에만 저장된다.

장점:

- 구현이 단순하다.
- 개인정보 수집이 거의 없다.

단점:

- 앱 삭제 시 기록이 사라진다.
- 기기 변경 시 기록이 이동되지 않는다.

1차 앱스토어 배포는 로컬 저장만으로도 가능하지만, 앱 설명이나 개인정보 문구에서 로컬 저장 중심임을 명확히 하는 것이 좋다.

### 19.4 접근성 보완 필요

일부 UIKit TextField에서 고정 폰트를 사용한다.

추후 확인할 것:

- 큰 글자 보기에서 레이아웃이 깨지지 않는지
- VoiceOver에서 주요 버튼이 자연스럽게 읽히는지
- 색상만으로 상태를 구분하지 않는지
- Reduce Motion 환경에서 과한 애니메이션이 없는지

### 19.5 탭 구조는 장기적으로 재검토 가능

현재 앱은 조건에 따라 하단 탭이 1개 또는 2개로 바뀐다.

사용자가 이전에 원해서 만든 구조지만, Apple 순정 앱 느낌을 더 강하게 가져가려면 장기적으로는 `TabView`를 제거하고 `NavigationStack` 기반으로 홈 목록에서 상세로 push하는 구조가 더 자연스러울 수 있다.

단, 이 변경은 UX가 크게 바뀌므로 사용자의 명확한 요청이 있을 때 진행하는 것이 좋다.
